Contributing to LPrint
======================

I'm happy to accept changes into LPrint.  For new drivers, I ask that
contributors provide me with a printer and supplies that would allow me to
debug any issues *and/or* commit to providing assistance on any issues that
are reported on GitHub.

Contributions should be submitted as pull requests on the
[LPrint GitHub project](https://github.com/michaelrsweet/lprint).  New code
MUST be supplied under the existing LPrint license (Apache 2.0).


Using AI Coding Tools
---------------------

The use of AI coding tools must be disclosed when you submit an issue or pull
request.  In the interest of standardization, please include the Linux Kernel
["Assisted-by"](https://docs.kernel.org/process/coding-assistants.html) tag in
the description:

    Assisted-by: AGENT_NAME:MODEL_VERSION [TOOL1] [TOOL2]

> *Note:* Purely AI-generated code cannot be accepted for legal and ethical
> reasons.  AI code contributions must be manually reviewed and submitted by
> a person with the legal capacity to provide such code under the terms of the
> LPrint license.
